# 招聘助手插件 · 产品设计文档

## 一、产品定位

面向主动求职的用户，在招聘网站职位详情页嵌入 AI 分析面板，提供
「JD 解析 → 逐项技能匹配 → 打招呼语生成 → 定制简历下载」全流程辅助。
参考 Jobright 插件交互风格，适配中文招聘场景。

**支持平台**：BOSS直聘 / 智联招聘 / 猎聘 / 拉勾

## 六、个人档案结构

**来源：由上传的简历 PDF 自动解析生成，用户确认后本地持久化存储。**
作为匹配分析和定制简历生成的基准数据，用户可在档案页随时编辑修改。

### 解析来源与存储字段

| 模块 | 字段 | 解析来源 | 示例 |
|------|------|----------|------|
| 基本信息 | 姓名、目标岗位、工作年限、期望薪资、联系方式 | PDF 解析 | 孔依依 / AI 产品经理 / 5年 |
| 教育背景 | 学校、学历、专业、时间 | PDF 解析 | 北京邮电大学 / 硕士 |
| 工作经历 | 公司、职位、时间、经历亮点（多条） | PDF 解析 | 多段带数字的亮点描述 |
| 原始简历 | PDF 提取的完整文本 | PDF 解析 | 作为定制简历的改写底稿 |

### 解析确认流程
```
用户上传 PDF
  └─ pdf.js 提取文字
       └─ DeepSeek 解析结构化字段（JSON）
            └─ 展示确认页：分模块展示解析结果
                 ├─ 用户可逐项编辑修正（解析偏差时）
                 └─ [确认保存] → 存入 chrome.storage.local
```

> 解析完成后，用户可随时通过插件设置页进入档案编辑。

## 七、完整交互流程

### 7.1 首次使用（Onboarding）

```
打开插件（未完成设置）
  └─ 引导页：一步完成
       └─ [上传简历 PDF]
            └─ pdf.js 提取文字 → DeepSeek 解析结构化档案
                 └─ 展示解析结果（分模块：基本信息 / 教育背景 / 工作经历）
                      ├─ 用户可编辑修正解析偏差
                      └─ [确认，开始使用] → 解锁主功能
```

> 设计原则：零手填，上传即建档，降低启动门槛。

### 7.2 职位分析主流程

```
打开支持平台的职位详情页
  └─ 插件自动检测到职位信息
       └─ 弹窗显示：[✦ 分析职位]
            └─ 点击 → 职位信息确认页
                 ├─ 自动填充：职位名称、公司名称、JD 全文
                 ├─ 用户可编辑任意字段
                 └─ [保存并分析]
                      └─ 分析中状态（✦ 动画 + "AI 分析中，约 10-20 秒"）
                           └─ 分析完成 → 结果页 ──────────────────┐
                                                                   ↓
```

### 7.3 分析结果页

```
┌─ 结果页 ──────────────────────────────┐
│                                       │
│  公司名称                    ( 80% )  │  ← 圆形匹配度 badge
│  职位名称                             │
│  刚刚分析                             │
│                                       │
│  ── 技能匹配详情 ──                   │
│  ✅ RAG / 向量数据库经验              │
│  ✅ LLM 评估体系设计                  │
│  ⚡ ToB 产品经验（有相关但非主线）    │
│  ❌ 商业化 / 定价策略                 │
│                                       │
│  [✦ 生成打招呼语]                     │
│  [✦ 生成定制简历]                     │
│                                       │
│  ── 投递反馈 ──                       │
│  投出去了吗？[已投递] [未投递]        │  ← 用于后续评分校准
│                                       │
└───────────────────────────────────────┘
```

### 7.4 生成打招呼语

```
点击 [✦ 生成打招呼语]
  └─ 弹窗内展开（inline，无需跳转）
       ├─ 版本 1（专业简洁）：...文字...  [复制]
       └─ 版本 2（热情积极）：...文字...  [复制]
```

- 字数控制 80-120 字
- 禁止"您好"/"我对贵公司很感兴趣"开头
- 至少引用一个与 JD 匹配的具体数字

### 7.5 生成定制简历

```
点击 [✦ 生成定制简历]
  └─ 弹窗内：加载状态（"正在根据 JD 定制你的简历..."）
       └─ 新标签页自动打开 → 简历预览页
            ├─ 展示四个模块（可点击任意段落直接编辑）：
            │    ├─ 基本信息
            │    ├─ 个人优势（AI 根据 JD 改写，技能融入其中）
            │    ├─ 工作经历（AI 改写措辞，突出 JD 匹配点）
            │    └─ 教育背景
            └─ [下载 PDF]  →  使用 html2pdf.js 生成并下载
```

**AI 改写逻辑（双层硬约束，防止捏造）：**

- 原始底稿：用户上传并确认的简历 PDF 提取文本，切分为语义片段建立本地索引
- **Layer 1（改写前）**：每条改写内容必须能在原始索引中检索到来源，无法溯源的内容不允许生成
- **Layer 2（改写后）**：对输出结果做语义相似度验证，低于阈值的句子自动 fallback 到原文
- 用户端呈现：每条改写内容旁显示"基于你的[XX公司经历]"，可点击查看原始依据

### 7.6 每日限额提示

```
用户当日分析次数达到上限（第 31 次）
  └─ 展示提示弹窗：
       "今日分析次数已用完，明天再来继续分析吧 ✦"
       └─ [我知道了]  →  关闭弹窗
```

### 7.7 投递结果回收（反馈钩子）

```
用户标记"已投递"后，5 天后插件提醒：
  └─ "你投递的[职位名称]有回音了吗？"
       ├─ [收到回复 ✓]
       ├─ [已读不回]
       └─ [未收到消息]
```

> 收集到足量数据后，用实际回复率校准匹配评分模型，验证"80分"是否真的预测了面试成功。

### 7.8 再次访问同一职位

```
再次打开已分析职位的详情页
  └─ 直接显示上次缓存的分析结果（无需重新分析）
       └─ 右上角显示"上次分析时间"
```

## 八、技术方案

| 模块 | 方案 |
|------|------|
| 插件类型 | Chrome Extension Manifest V3 |
| 语言 | HTML / CSS / JavaScript（无框架） |
| AI 接口 | DeepSeek API（deepseek-chat） |
| PDF 文字提取 | pdf.js |
| PDF 生成下载 | html2pdf.js |
| 数据持久化 | chrome.storage.local |
| JD 提取 | content.js 启发式规则（见下方） |

### JD 提取策略（防御性方案）

不依赖各平台固定 class name，采用启发式规则定位 JD 内容，避免平台改版导致功能失效：

```
启发式识别规则（优先级从高到低）：
  1. 文本长度最长的 <div> / <section> 块（> 200 字）
  2. 包含高密度职位关键词（"职责" / "要求" / "技能" / "经验"）的文本块
  3. 页面标题附近的主要内容区域

容错机制：
  └─ 提取结果置信度 < 阈值时，展示"请手动粘贴 JD"输入框
       └─ 用户手动粘贴后保存，下次访问同平台自动优先使用人工确认的区域
```

### 隐私合规处理

> 对应《个人信息保护法》第 17 条知情同意要求

- **Onboarding 授权确认**：上传简历前展示一句话说明："你的简历内容将发送至 AI 服务进行解析，[查看隐私政策]"，用户点击确认后方可继续
- **数据脱敏**：简历文本发送至 DeepSeek API 前，自动脱敏姓名、手机号、邮箱（正则替换为 `[姓名]` / `[联系方式]`）
- **本地优先**：所有个人数据仅存储于 `chrome.storage.local`，不上传至任何第三方服务器

### DeepSeek Prompt 设计（四步）

**Step 0 · 简历解析（Onboarding 时执行一次）**
```
输入：pdf.js 提取的简历原文（脱敏后）
输出：JSON
{
  "basic": { "name": "孔依依", "target_role": "AI产品经理", "years": 5, "salary": "25-40K", "contact": "..." },
  "education": [{ "school": "北京邮电大学", "degree": "硕士", "major": "...", "period": "..." }],
  "experiences": [{ "company": "...", "role": "...", "period": "...", "highlights": ["..."] }]
}
```

**Step 1 · JD 解析**
```
输入：JD 原文
输出：JSON
{
  "must_have": ["技能1", "技能2"],
  "nice_to_have": ["技能3"],
  "responsibilities": ["职责1", "职责2"]
}
```

**Step 2 · 匹配分析**
```
输入：个人档案 + Step1 解析结果
输出：JSON
{
  "score": 80,
  "matches": [
    { "skill": "RAG架构", "status": "match", "evidence": "主导法考RAG系统，召回准确率97%" },
    { "skill": "商业化经验", "status": "missing", "evidence": null }
  ]
}
```

**Step 3 · 简历改写**
```
输入：原始简历语义片段索引 + JD must_have 列表
输出：改写后的简历各模块文本（JSON 结构），每条内容附带来源片段 ID
{
  "strengths": [
    { "text": "亮点1", "source_id": "exp_0_highlight_2" },
    ...
  ],
  "experiences": [{ "company": "...", "role": "...", "highlights": [{ "text": "...", "source_id": "..." }] }]
}
```

> 分数由结构化数据按权重计算，不依赖 LLM 直接打分（防止打分漂移）。
> 改写结果由 source_id 回溯验证，无法溯源则 fallback 原文（防止捏造）。

## 九、作品集展示亮点

| 设计决策 | 体现的 PM 能力 |
|----------|--------------|
| 两步 Prompt（解析 → 匹配） | Chain-of-Thought 拆解，降低幻觉 |
| 分数由结构化数据计算，非 LLM 直接打 | 可解释评分架构 |
| must-have / nice-to-have 权重差异 | 需求优先级判断，类比 Gate 机制 |
| 匹配 evidence 字段 | 结果可追溯，防止黑盒 |
| 改写双层硬约束（source_id 溯源 + 相似度验证） | AI 护栏设计，技术手段保障内容可信 |
| 同职位缓存分析结果 | 用户体验细节，减少重复等待 |
| 上传简历自动建档，零手填 | 降低启动门槛，减少用户摩擦 |
| 投递结果反馈钩子 | 闭环数据收集，用真实转化率校准评分 |
| 每日 30 次免费额度设计 | 前期优先积累用户，DeepSeek 成本低，限额留有后续商业化升级空间 |
| 启发式 JD 提取 + 手动兜底 | 防御性技术方案，平台改版不导致产品宕机 |
| PIPL 合规脱敏处理 | 数据安全意识，面向真实上线的产品思维 |
| 北极星指标定义（HR 回复率） | 以用户目标达成而非功能上线衡量产品价值 |
